# NN.md — custom neural-net faction driver

> `ai/net/` provides `Net` / `NetArch` / `NetPolicy` / `Evolution` (all tested + deterministic); a full 25-arch
> champion library is baked and beats the baseline. Regenerate it headless — `scripts/bake-champs.sh` (gen-1 vs
> the heuristic) then `scripts/train-champs.sh` (gen-2 NN-vs-NN) — or **train one arch live** in the **Train NN**
> screen (opened from the menu / the BRAINS-tab link) and watch any net **think** in the **BRAINS** tab.

## The idea (the original "Q-gress")

Each faction is driven by a small purpose-built network whose **output layer _is_ the 17 behaviour
sliders** (10 `QActions` + 7 `QDestinations`, each 0..1). Input is a normalized `Observation` of world
state; output is a `SliderVector`, re-evaluated at a slow cadence (**once per scoring checkpoint**). The
slider vector stays the action substrate — the net does **not** replace per-agent `ActionSelector`.

## What's built (`ai/net/`)

- **`Net`** — a small MLP: `Observation.SIZE` (13) inputs → the hidden layers → `SliderVector.SIZE` (17)
  outputs (sigmoid → 0..1). All weights in one flat genome array; pure deterministic forward pass, with a
  `forwardTraced` variant that returns per-layer activations for the visualizer.
- **`NetArch`** — the architecture: any number/width of hidden layers + bias + hidden `activation`
  (default **two hidden layers of 16**, tanh). The trainer offers two width dropdowns (**4 / 8 / 16 / 24 /
  32** each → 4×4 … 32×32). `genomeSize()` / `layerOffset()` make the flat genome arch-aware.
- **`NetPolicy`** — wraps a `Net` as a `FactionPolicy`: live `Observation` → `SliderVector`, cached and
  re-evaluated **once per checkpoint** (installing it costs no more per-action than a slider read).
- **`Evolution`** — a resumable `(μ+λ)` GA (elitism + gaussian mutation; the env is non-differentiable and
  the reward episodic, so ES, not gradient RL). Fitness = mean **checkpoint-win margin** (net checkpoints led
  across a full cycle, summed MU margin as a sub-integer tiebreak — `MatchResult.checkpointFitness`, matching
  `CheckpointStats`' cycle-winner rule) over K seeded `SimRunner` matches, each pinned to the **shipped default
  balance** (`MatchSetup.useDefaultBalance`, ignoring the player's live menu sliders) so champions train against
  one canonical target. Its RNG (a trainer-local instance) is independent of the global `Rng` (which `SimRunner`
  reseeds per match) → fully deterministic given (grid, seed, config). `Evolution.Session.stepGenome()` scores one
  genome at a time so a UI can show live progress; `train()` loops it; `Session.elites()` exposes the pool a bake
  selects its champion from by **held-out** fitness.
- **`GenomeIO` / `NetStore` / `Champion` / `ChampionLibrary`** — JSON encode/decode of genome + arch + fitness;
  `localStorage` persistence (survives reload); a **per-arch baked champion library** (one JSON file per
  `NetArch` under `resources/champions/` + an `index.json` manifest, bundled via `require`, regenerated by the
  scripts below), with `Champion` the default-arch alias. A single champion is the shareable unit —
  download/upload one arch's genome; a loaded champion (`ChampionLibrary.installChampion`) persists per arch and
  is tagged with its **provenance** (filename + load time), so the UI shows a repo default vs a user net and
  offers a per-arch **revert** (`revertToBundled`). `NetStore.loadNet(arch)` installs the chosen arch's champion;
  onboarding/DriverControls pick it via **two per-layer width dropdowns** (or **Rnd**, seed-resolved so shared
  links reproduce).
- **Rebake scripts** — matches run a **full scoring cycle** (`Config.ticksPerScoringCycle`, ~35 checkpoints; the
  fitness goal is "win the cycle", so it can't be judged in less). `scripts/bake-champs.sh` trains each arch's
  champion vs the adaptive `HeuristicPolicy` baseline (gen 1); `scripts/train-champs.sh` then trains a fresh
  challenger per arch **vs that arch's current champion** (NN-vs-NN, gen 2+), overwriting a per-arch JSON only
  when a challenger wins a round-robin decider, and prints a cross-arch overall ladder. Both run headless via
  `jsNodeTest` (`ai.net.ChampionBake` / `ChampionTrainNN`), one arch per fresh Node process; the champion is
  selected by **held-out** (unseen-seed) fitness so it never ships an overfit net.

## Where to see it

- **Train NN screen** (`system/ui/panel/TrainerPanel`) — opened from the menu / the BRAINS-tab link as its own
  full-screen overlay with the live game **paused** behind it. Pick population / generations / mutation /
  per-layer widths / activation + an **opponent** (that arch's current champion by default, or a **loaded** net),
  hit **Train**: one genome per `setTimeout` (UI never blocks), a fitness curve **vs the opponent**, and the
  selected arch's champion shown as an activation diagram + genome up front (repainted per arch). A **held-out
  decider** (unseen seeds, both colours) then reports whether the challenger beats the opponent and tags it with
  a **vs-baseline** fitness (comparable to the repo champions). **Save** / **Install** / **Download** / **Load**
  the JSON. A second section runs a **leaderboard** (round-robin over the live grid). Bracketed by `HeadlessRun`
  (snapshot + full pause + silenced FX) so the live game is untouched.
- **BRAINS tab** (`system/ui/panel/BrainsPanel` + `system/ui/panel/NetVizPanel`) — per faction: the **live activation
  diagram** (layers as node columns, edges lit by contribution, the top output's "chosen path") above the
  **genome heatmap**, plus arch / fitness / driving-input / peak-hidden / favoured actions.

## Mechanism (decided)

- **Neuroevolution / ES**, not tabular Q-learning or gradient RL: non-differentiable env, sparse/episodic
  reward (checkpoint-win margin), continuous action space → fast headless match → fitness number, embarrassingly
  parallel. The "Q" in Q-gress is heritage branding; the mechanism is honest ES.

## Open questions / next
- ES variant (CMA-ES vs the current GA), and a fuller self-play **league + Hall-of-Fame**. `train-champs.sh`
  already does gen-over-gen NN-vs-NN (each challenger trains vs its own incumbent) — the first step past the
  fixed `HeuristicPolicy` baseline — but not yet a rotating opponent pool / league.
- **Fitness shaping**: checkpoint-win margin already discourages one-blowout farming (you must *lead* the most
  checkpoints), but could still be shaped to reward *interesting* play (lead changes / field layering).
  (Recruiting is no longer a lever here: it's a capped idle-fallback, not one of the 17 sliders.)
- A **library of trained nets per `NetArch`** — **shipped**: all 25 archs are baked (`ChampionLibrary`, one JSON
  file per arch under `resources/champions/`, regenerated by `bake-champs.sh` / `train-champs.sh`), picked per
  side in onboarding/DriverControls. Each beats the `HeuristicPolicy` baseline on held-out seeds. Next: deeper
  budget / a real self-play league to strengthen them.
- Real line-coverage once the functional core moves to `commonMain` + a `jvm()` test target.
